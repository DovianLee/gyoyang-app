# 교양 앱 프로젝트 — 현황 (Claude chat 동기화용)

> **이 문서 용도:** 이도규(도비안)가 Claude Code로 진행 중인 프로젝트의 전체 맥락 요약.
> Claude chat(claude.ai)에 통째로 붙여넣어 상황을 공유하기 위한 문서. 최종 수정: 2026-06-24.

## 1. 한 줄 요약
"매일 아침 7시, **웹검증된 교양 지식 3개**를 폰으로 받는 **PWA 앱**"을 직접 만들며 코딩을 배우는 프로젝트. 최종적으로 공개·제품화(사용자의 '감정치유 제품 꿈' 연습)까지 염두에 둠.

## 2. 만드는 사람 · 작업 방식
- **코딩 초보.** '배우면서 만들기' 모드 — Claude가 뼈대 코드를 쓰고 한 줄씩 설명, 본인이 읽고 고치고 질문.
- 정밀한 언어와 결정 사전 승인을 선호. 모호한 표현은 짚어주길 원함.
- **핵심 통점: '축적 > 출력'** (입력만 쌓고 못 꺼냄). → 규칙: **한 번에 한 마일스톤, 각 단계는 반드시 '작동하는 결과물'로.**
- **워크플로우:** Claude Code가 결정·검증·알아둘 항목을 `[확인요청]`으로 띄우면, 사용자가 이 문서와 함께 Claude chat에 물어 답을 가져옴. 마일스톤마다 검토 체크포인트.

## 3. 콘텐츠 설계 (지식 브리핑) — 확정
- **분야(6, 하이브리드):** 됨됨이·관계 / 경제 / 역사 / 과학 / 철학(윤리 포함) / 종교. 선택은 '안 겹치는 셔플', 하루 믹스/단일 랜덤.
- **선정 우선순위:** ① 사회·대화에서 쓰임 ② 교과서적 필수 를 자주 번갈아 + ③ 흥미는 양념으로.
- **항목 구조(층) — 최종 확정:** 겉=기본(분야·제목·한 줄 정의·직관 고리) / 속=심화('더보기': 유래·원리·예시·⚠️흔한 착각·의의·활용·더 배우기·✓웹검증). 완성 템플릿+예시 = `CONTENT-TEMPLATE.md`.
- **분량:** 하루 3개.
- **정확성:** '무거움' = 매 항목 웹검증 후 전달, 불확실하면 표시. 시대 안 타는 정설 위주(최신 뉴스·수치 회피).
- **복습:** 일요일 = 시험(객관식 + 주관식), 답하면 채점·해설. (복습 + 레벨 보정 겸)
- **무너진 날:** 죄책감·스트릭 압박 없음.

## 3-1. 레벨 · 학습 엔진 — 확정
- **레벨 사다리(분야별):** L0 모름 → L1 인지 → L2 설명 → L3 적용 → L4 통찰. (= '잘 배움' 4기준). 콘텐츠는 '현재 레벨 → 한 칸 위' 겨냥.
- **레벨 측정:** 적응형 진단(2~3문항, L3부터 맞으면↑·틀리면↓). **타당도 규칙:** 객관식은 상식·추론으로 우회됨 → '지식'은 주관식(개방형 회상)으로, 각 문항은 '개념 모르면 못 푸는' 것이어야 함. 추론력 ≠ 개념지식(별개 측정). 보강: 레벨당 2문항·확신도·시간 누적.
- **사용자 프로필(진단):** 추론·상식 高 / 교양 용어·개념 低 → 콘텐츠 방향 = '직관에 이름·구조 붙이기'.
- **단계적 고도화 (사용자 동의):**
  - **v1 (현재):** L1~4 + 적응 진단 + 단순 일요 시험.
  - **v2:** 간격 반복(틀리면 짧게·맞으면 길게 재등장) + 카드 끝 "오늘 써먹기" 한 줄.
  - **v3:** 기억 강도(이해깊이 × 기억강도 2축) + 개념 연결망 + "네 말로 설명해봐".
