# SESSION-GUIDE — 세션 운영 가이드

## 0. 이 프로젝트의 파일 구조 (각 파일의 역할)
- index.html         : 실제 앱. 교양 카드의 진짜 내용은 여기에만 둔다.
- PROJECT-STATUS.md  : 현재 상태·설계·로드맵·다음 할 일. 자주 읽는 문서라 가볍게 유지.
- CHANGELOG.md       : 날짜별 변경 내역(과거 기록). 길어져도 됨.
- CONTENT-TEMPLATE.md: 교양 카드 양식 + 웹검증 범위 기준.
- SESSION-GUIDE.md   : 이 문서. 세션 운영·문서·push·병렬 규칙.

## 1. 세션 분리 기준
- 한 세션 = 한 매듭(예: "카드 1개 완성"). 매듭이 끝나면 세션을 닫는 걸 기본으로 한다.
- 컨텍스트가 60%에 도달하면, 매듭 전이라도 그 자리에서 저장하고 세션을 닫는다(사용자가 판단).
- 마일스톤(M1, M2...)이 바뀔 때 새 세션을 연다.

## 2. 세션 닫을 때 (마지막에 할 일)
1. 이번 세션 작업을 PROJECT-STATUS.md에 반영(상태·현재 위치 갱신, 최종 수정 날짜 포함).
2. 변경 내역은 CHANGELOG.md에 날짜별로 기록(상세형).
3. 위 둘을 git commit. 4. git log --oneline로 확인.
- ※ 교양 카드 실제 내용은 index.html에만. 다른 문서에 복사하지 않는다.

## 3. 세션 열 때
### Claude Code 새 세션:
1. PROJECT-STATUS.md를 읽는다(현재 위치·다음 할 일).
2. SESSION-GUIDE.md도 읽어서 이 운영 규칙을 따른다.
3. CHANGELOG.md와 git log --oneline로 흐름을 본다.
4. 문서 상태와 실제 파일이 일치하는지 가볍게 대조한다(전체 스캔 말고 핵심만).
   어긋나면 진행 말고 사용자에게 알린다.
5. 요약을 보여주고, 바로 작업 말고 사용자 확인을 기다린다.
### claude.ai 새 채팅(전략·검토용):
- PROJECT-STATUS.md와 SESSION-GUIDE.md를 붙여넣어 맥락을 전달한다.
- 긴 과거 대화는 옮길 필요 없다(결론은 파일에 있음).

## 4. 문서 갱신 규칙 (항상)
- 변경이 생기면: 최신 상태는 PROJECT-STATUS.md에(날짜 포함), 변경 내역은 CHANGELOG.md에.
- PROJECT-STATUS.md 갱신은 Claude Code만 한다. claude.ai 채팅은 읽고 참고만.

## 5. push(배포) 규칙
- push는 자동화하지 않는다. 커밋은 자유롭게, push는 "라이브에 올릴 매듭"에서 의도적으로.
- push는 사용자가 직접 터미널에서 실행한다.
- push하면 Vercel이 자동 재배포. 배포 진행은 Claude Code 담당, 확인 안 되면 추측 말고 문의.
- 비밀·API 키는 절대 커밋하지 않는다(.gitignore + 환경변수 사용). 특히 M6에서 Claude API 키 사용 시 주의.

## 6. 병렬 작업 계획 (M5 이후)
- 지금은 직렬(한 세션씩 바통터치). 코드가 index.html 한 파일에 모여 병렬은 충돌 위험.
- M5(데이터 분리)에서 파일이 갈라진 뒤 병렬 도입.
- 병렬 절대 규칙: 세션마다 만지는 파일이 겹치지 않게. 겹치면 덮어쓰기 사고.
